# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Purpose

AI Support Triage Agent ‚Äî a portfolio project demonstrating AI Enablement skills. The system accepts freeform customer support ticket text and runs it through a single Claude-powered agent that:

1. Classifies **urgency** (high / medium / low), **category** (bug / billing / feature / onboarding), and **sentiment**
2. Detects whether the ticket represents an **AI automation opportunity**
3. Generates a **draft response** with a **confidence score** (0.0‚Äì1.0)
4. Traces every LLM call to **Langfuse** (latency, token counts, scores)
5. Accepts **user feedback** from the Streamlit UI and sends it back to Langfuse as a score event

## Folder Conventions

- **`/app`** is the only place that imports `anthropic` or `langfuse`. The UI layer must never call the LLM directly.
- **`/ui`** communicates with the backend exclusively over HTTP (`httpx` calls to the FastAPI service). No shared Python modules between `app/` and `ui/`.
- **`/tests`** mirrors the `app/` structure. Tests should mock the Anthropic client and Langfuse client ‚Äî never hit live APIs in tests.
- All new backend modules go inside `/app`. Do not create top-level Python files.

## Data Model Rules

All models live in `app/models.py`. Follow these rules:

- **`TicketRequest`** ‚Äî the inbound payload: `ticket_id: str`, `text: str`. No other fields; the agent derives everything else.
- **`TriageResult`** ‚Äî the full output: `ticket_id`, `urgency` (enum), `category` (enum), `sentiment` (enum), `automation_opportunity: bool`, `draft_response: str`, `confidence: float`, `trace_id: str`.
- **`FeedbackRequest`** ‚Äî user feedback payload: `trace_id: str`, `score: float` (0.0‚Äì1.0), `comment: str | None`.
- Use **Python enums** (not string literals) for `Urgency`, `Category`, and `Sentiment` ‚Äî this ensures the LLM output is validated against a closed set before returning.
- All models use **`model_config = ConfigDict(use_enum_values=True)`** so enums serialize as strings in JSON.
- Confidence is always a `float` clamped to `[0.0, 1.0]`. Validate with `@field_validator`.
- `trace_id` on `TriageResult` is populated by the agent after the Langfuse trace is created ‚Äî it is never generated by the API layer.

## Prompt Management Rules

All prompt strings live exclusively in `app/prompts.py`. Rules:

- **No f-strings or string concatenation inside `agent.py`**. Prompts are functions that accept parameters and return a fully-formed string.
- Each prompt function is named `build_<purpose>_prompt(...)`, e.g. `build_triage_prompt(ticket_text: str) -> str`.
- The system prompt and user prompt are kept **separate**: `build_system_prompt() -> str` returns the static system instructions; `build_triage_prompt(ticket_text)` returns the user turn.
- Prompts must instruct the model to respond with **valid JSON only** ‚Äî no markdown fences, no preamble. The agent parses with `json.loads()` and catches `JSONDecodeError`.
- The JSON schema expected from the model must be documented as a comment directly above the prompt function that requests it.
- When changing prompt logic, update the schema comment in the same commit.

## Langfuse Tracing Rules

Langfuse instrumentation lives entirely in `agent.py`. Rules:

- Every call to `anthropic.messages.create(...)` must be wrapped in a **Langfuse generation**: create the generation before the call (record input/model/params), update it after (record output, token usage, latency).
- The **trace** wraps the full `triage()` function invocation. The trace `name` is `"ticket-triage"`, the `input` is the raw ticket text, and the `output` is the serialized `TriageResult`.
- Always capture: `model`, `usage.input_tokens`, `usage.output_tokens`, `latency_ms`.
- The `trace_id` returned by Langfuse must be stored and returned to the caller inside `TriageResult.trace_id` so the UI can send feedback referencing it.
- **Feedback** from the UI hits `POST /feedback`. The handler calls `langfuse.score(trace_id=..., name="user-feedback", value=score, comment=comment)` ‚Äî nothing else. No LLM calls in the feedback path.
- Never log raw ticket text to stdout in production; use Langfuse trace metadata for that.
- Langfuse client is initialized once at module load in `agent.py` using env vars `LANGFUSE_PUBLIC_KEY`, `LANGFUSE_SECRET_KEY`, `LANGFUSE_HOST`.

## Docker Rules

- `docker-compose.yml` defines two services: `api` (port 8000) and `ui` (port 8501). Langfuse runs separately on the host at port 3000 ‚Äî it is not managed by Compose.
- The `api` service builds from `./Dockerfile`; the `ui` service builds from `./ui/Dockerfile`.
- Containers reach the host-machine Langfuse via `host.docker.internal:3000`. The `LANGFUSE_HOST` env var is overridden in `docker-compose.yml` to use this address; the `.env` file keeps `localhost:3000` for local dev outside Docker.
- `extra_hosts: [host.docker.internal:host-gateway]` is set on both services for Linux Docker compatibility.
- The `api` service declares a `healthcheck`; the `ui` service uses `depends_on: api: condition: service_healthy`.
- The Streamlit container reads `API_BASE_URL=http://api:8000` (set in `docker-compose.yml`) to reach the API by Compose service name ‚Äî never hardcode `localhost` for inter-service calls.
- Never bind-mount source code into the container ‚Äî rebuild on change.
- `ANTHROPIC_API_KEY` is never hardcoded anywhere. It is read from `.env` at runtime.

## Coding Style

- **Python 3.11+** ‚Äî use `match/case` for enum dispatch, `X | Y` union syntax, `tomllib` if needed.
- **Ruff** for linting and formatting (replaces black + flake8 + isort). Config lives in `pyproject.toml`.
- FastAPI route handlers are thin: they validate input, call into `agent.py`, and return the result. No business logic in `main.py`.
- `agent.py` contains one public async function `triage(ticket: TicketRequest) -> TriageResult`. Internal helpers are prefixed with `_`.
- Use `async def` throughout the FastAPI app. The Anthropic SDK async client (`AsyncAnthropic`) is used in `agent.py`.
- Raise `HTTPException` only in route handlers, not inside `agent.py`. Agent functions raise domain exceptions (e.g., `TriageError`) that handlers catch and convert.
- Type-annotate every function signature. No `Any` unless wrapping an untyped third-party return.
- Keep functions under ~40 lines. If a function is growing, split parsing logic from LLM call logic from Langfuse logging logic.
